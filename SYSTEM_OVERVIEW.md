# みんなのAIゲームギャラリー — システム概要

> 子どもたちが生成AIを使って作ったゲームを、体験イベントの場でも、あとからでも楽しめるWebプラットフォーム。

---

## 1. プロジェクトの目的

本システムは、**生成AI体験イベント**で子どもたちが制作したゲーム作品を、次の2つの文脈で提供することを目的としています。

| 文脈 | 想定シーン | ユーザーの行動 |
|------|-----------|---------------|
| **作品展示（ギャラリー）** | イベント後・SNS共有・保護者への報告 | 開催地 → 開催日 → 作品一覧 → 遊ぶ |
| **体験当日（ライブ）** | 会場のプロジェクター・タブレット | すぐにゲームを選んで遊ぶ・素材を取得する |

現状は主に「ギャラリー」として設計・実装されています。ユーザーからの要望として、**体験当日にみんなで遊ぶ画面**と**素材ダウンロード画面**の追加が検討されています。

---

## 2. 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロントエンド | Vite 7 + Vanilla JavaScript（Hash SPA） |
| バックエンド / DB | Supabase（PostgreSQL） |
| ファイルストレージ | Supabase Storage（`game-files` バケット） |
| 認証 | Supabase Auth（管理者のみ） |
| フォント | M PLUS Rounded 1c / Noto Sans JP |
| 開発サーバー | `npm run dev` → http://localhost:3000 |

## 4. データモデル

```
locations（開催地）
  └── events（開催日）
        └── sessions（回・時間帯）  ※管理用。公開UIでは省略
              └── games（ゲーム作品）
                    ├── title          作品名
                    ├── storage_path   Storage上のフォルダパス
                    ├── entry_file     エントリHTML（デフォルト: index.html）
                    ├── is_published   公開フラグ
                    └── sort_order     表示順
```

**初期データ（setup.sql）**
- 開催地: 那覇市開催 / うるま市開催
- 各開催地にイベント・セッション・ゲームを管理者が追加

## 5. 画面一覧（現状）

### 5.1 公開サイト

| # | ルート | 画面名 | 機能 |
|---|--------|--------|------|
| 1 | `#/` | ホーム | 開催地を選択（背景画像付きカード、作品数表示） |
| 2 | `#/location/:id` | 開催地 | 開催日一覧（日付・作品数） |
| 3 | `#/event/:id` | 開催日 | ゲーム一覧（iframe静止プレビュー、スクリプト除去） |
| 4 | `#/play/:id` | プレイ | ゲームをフルサイズiframeで実行、フルスクリーン対応 |

**補足:** `session.js`（回・時間帯ごとの一覧）は実装済みだが、ルーターに未登録。公開UIでは開催日直下に全ゲームをフラット表示している。

### 5.2 管理画面（`/admin.html`）

| 機能 | 内容 |
|------|------|
| ログイン | メール + パスワード（Supabase Auth） |
| 開催地管理 | 追加・編集・画像アップロード |
| 開催日管理 | 追加・編集・削除（アコーディオンUI） |
| ゲーム管理 | フォルダD&Dアップロード、公開/非公開切替、編集・削除 |
| セッション | アップロード時に自動作成（`デフォルト` ラベル） |

## 6. 主要な処理フロー

### ゲームのアップロード（管理者）

1. 管理者がHTML/CSS/JSを含むフォルダをドラッグ＆ドロップ
2. `uploadGameFolder()` が Supabase Storage へファイル単位でアップロード
3. `games` テーブルにメタデータ（タイトル、storage_path 等）を登録
4. `is_published = true` で公開

### ゲームの表示（訪問者）

1. Supabase からゲームメタデータを取得
2. Storage から `index.html` を fetch
3. `<base href="...">` を注入して相対パスを解決
4. iframe の `srcdoc` に注入して表示
   - **一覧プレビュー:** `<script>` タグを除去 → 静止画風表示
   - **プレイ画面:** スクリプト有効、`sandbox="allow-scripts allow-same-origin allow-popups"`

## 8. UI/UX 評価（プロのウェブデザイナー視点）

### 8.1 総合評価

| 観点 | 評価 | コメント |
|------|------|---------|
| ビジュアルアイデンティティ | ★★★★☆ | パステルグラデーション・丸ゴシックで「子ども向けAI体験」の世界観が一貫 |
| 情報設計（IA） | ★★★☆☆ | 階層は明快だが、ギャラリー/ライブ/素材の用途分離がない |
| 操作性（子ども向け） | ★★★☆☆ | カード選択は直感的。ただし作品名非表示・戻る導線のみ |
| アクセシビリティ | ★★☆☆☆ | `role="button"` はあるがキーボード操作・フォーカス管理が不完全 |
| モバイル対応 | ★★★★☆ | 768px / 480px ブレークポイントあり。タッチターゲットは概ね良好 |
| パフォーマンス | ★★★☆☆ | 一覧でiframe多数生成。作品数増加時に負荷懸念 |
| 管理画面UX | ★★★★☆ | D&Dアップロード・公開切替が実用的。一般管理者向けに十分 |

---

### 8.2 良い点（維持すべきデザイン判断）

1. **統一されたデザインシステム**
   - CSS Variables によるカラーパレット（桜・ラベンダー・スカイブルー）が整理されている
   - グラスモーフィズムカード、ふわふわ背景パーティクル、ウェーブ装飾で「かわいい」トーンを維持

2. **開催地カードのビジュアル**
   - 背景画像 + オーバーレイ + 作品数バッジは、イベントの規模感を直感的に伝える

3. **ゲームプレビューの静止表示**
   - スクリプト除去によるサムネイル風プレビューは、ギャラリー用途として賢い選択
   - ホバー時の再生アイコンオーバーレイは「押したら遊べる」ことを明示

4. **管理画面の実用性**
   - アコーディオン + D&Dアップロード + 公開/非公開トグルは、当日運営に必要な機能が揃っている

5. **ローディング・空状態の配慮**
   - 「よみこみ中...」「まだ作品が登録されていません」など、子どもにもわかる日本語メッセージ

### 8.3 改善が必要な点

#### 🔴 優先度：高

| # | 問題 | 現状 | 改善提案 |
|---|------|------|---------|
| 1 | **作品名が一覧に表示されない** | プレビューカード下部は「▶ あそぶ」のみ | 作品タイトル・制作者名を常時表示。サムネイル下に2行テキスト |
| 2 | **用途モードの混在** | ギャラリー専用の4階層ナビのみ | トップに「作品を見る」「今日の体験」「素材ダウンロード」の3入口を設置 |
| 3 | **パンくずリスト不在** | 「← もどる」リンクのみ | `トップ > 那覇市 > 3月15日 > ゲーム名` のパンくずを追加 |
| 4 | **デフォルト画像が欠落** | `/images/naha.png` 等を参照するが `public/images/` が空 | 画像アセットの配置、または Supabase 上の `image_url` 運用を徹底 |

#### 🟡 優先度：中

| # | 問題 | 現状 | 改善提案 |
|---|------|------|---------|
| 5 | **セッション階層の不整合** | DB上は sessions があるがUI上は省略 | 複数回開催がある場合は `#/session/:id` をルーターに復活させる |
| 6 | **検索・フィルターなし** | 作品数が増えると一覧が長くなる | 作品名検索、タグ（ジャンル）フィルターを追加 |
| 7 | **iframe パフォーマンス** | イベントページで全作品のiframeを同時生成 | Intersection Observer による遅延読み込み、またはサムネイル画像への切替 |
| 8 | **プレイ画面の没入感** | ヘッダー + iframe の二段構成 | 会場モードでは UI クロームを最小化。ゲーム終了時の「別の作品を遊ぶ」導線 |
| 9 | **SNSシェア未対応** | OGP メタタグはトップページのみ | 作品ごとの OGP（`/play/:id` 用）と「この作品をシェア」ボタン |
| 10 | **404 / エラー画面** | シンプルなテキストのみ | イラスト付き + 「似た作品を探す」リンク |

#### 🟢 優先度：低（将来）

| # | 問題 | 改善提案 |
|---|------|---------|
| 11 | キーボード操作 | Enter/Space でカード選択、Tab フォーカスの可視化 |
| 12 | ダークモード | 不要（子ども向け明るいトーンを維持） |
| 13 | 多言語 | 沖縄イベントなら英語/やさしい日本語の切替も検討 |
| 14 | アニメーション過多 | `prefers-reduced-motion` メディアクエリで背景アニメーションを抑制 |

### 8.4 体験当日モード — UI/UX 設計提案

会場での利用を想定した **Live Mode** の設計指針:

```
┌──────────────────────────────────────────────┐
│  🎮 今日の体験 — 3月15日 那覇市              │
│                                              │
│  ┌────────┐  ┌────────┐  ┌────────┐        │
│  │        │  │        │  │        │        │
│  │ 作品A  │  │ 作品B  │  │ 作品C  │        │
│  │        │  │        │  │        │        │
│  └────────┘  └────────┘  └────────┘        │
│   田中くん     佐藤さん     山田くん           │
│                                              │
│  [ 🎲 ランダムで遊ぶ ]  [ 📦 素材をDL ]      │
└──────────────────────────────────────────────┘
```

**設計原則**
- **タップ数最小化:** トップ → ライブモード → ゲーム開始 = 2タップ
- **大きなタッチターゲット:** 最小 48×48px、推奨 64×64px 以上
- **高コントラスト:** プロジェクター投影を想定し、文字サイズ 18px 以上
- **作品への称賛:** 制作者名を目立たせ、体験の達成感を演出
- **QRコード入口:** 会場に掲示する QR から `#/live/:eventId` へ直接遷移

### 8.5 素材ダウンロード画面 — UI/UX 設計提案

```
┌──────────────────────────────────────────────┐
│  📦 素材ダウンロード                          │
│  「宇宙冒険ゲーム」の制作素材                  │

│                                              │
│  [ 📁 すべてZIPでダウンロード ]               │

│  💡 この素材を使って、家でもゲームを          │
│     作ってみよう！                            │
```

**設計原則**
